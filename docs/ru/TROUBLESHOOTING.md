# Устранение неполадок

## `NIRI_SOCKET is not set`

Утилита должна запускаться **внутри сессии niri**, где выставлена переменная `NIRI_SOCKET`. Проверка:

```sh
echo "$NIRI_SOCKET"
```

Если пусто, вы не под niri или сессия не экспортирует переменную (редко). Запускайте из терминала на niri.

## Ошибки IPC / несовпадение версий

Сообщения вида «unexpected IPC response» или ошибки niri при `Action` часто связаны с **несовпадением версии** niri и крейта `niri-ipc`, с которым собран `niri-session-manage`. Соберите проект с тем же `niri-ipc`, что соответствует вашему niri (`niri --version`), или установите niri из того же релиза, что и зависимость в `Cargo.toml`.

## `--graceful-shutdown` закрыл все окна

Команда **`niri-session-manage --graceful-shutdown`** после сохранения JSON вызывает IPC **`CloseWindow`** для каждого окна: без подтверждения и без отличия «важных» приложений. Несохранённые данные в приложениях могут быть потеряны — используйте осознанно (например перед `niri msg action quit` или выходом из сессии).

## Медленный или «рваный» `--load`

Загрузка **не ждёт** появления окон: процессы стартуют подряд с паузами из **`[load]`** / CLI. Если niri или диск не успевают, увеличьте `ipc_settle_ms` и при необходимости `spawn_start_delay_ms` (или `--ipc-settle-ms`, `--spawn-start-delay-ms`). См. [LOAD_RESTORE.md](LOAD_RESTORE.md).

При ошибке **запуска** процесса (или отсутствии правила `[[launch]]`) в логе будет `окно пропущено`; при включённых уведомлениях — ещё `notify-send` (нужен `libnotify`).

## Пустой или битый JSON

- Убедитесь, что файл сохранён полностью и это валидный JSON.
- Поле `schema` должно быть знакомым для версии `niri-session-manage`.

## Пустой список окон после `--save`

В сессию попадают только окна с известным **PID** и непустой командной строкой в `/proc`. Окна без PID (например, часть порталов) в MVP не сохраняются.

## X11-приложения (Chrome и др.) и `xwayland-satellite`

В niri X11-клиенты идут через **xwayland-satellite**. В сессии в JSON может сохраниться `command` такого вида:

```text
xwayland-satellite :1 -listenfd …
```

Это **не** команда для повторного запуска приложения (дескрипторы сессионные). При **`--save`** такие строки остаются в JSON вместе с `app_id` и `title` — раскладка сохраняется. При **`--load`** для таких строк нужен **TOML-конфиг** `[[launch]]`: `app_id` / `title_contains`, поле **`resolve`** (`-listenfd` или basename вроде `xwayland-satellite`) и реальная `command`. См. [CONFIG.md](CONFIG.md).

Если правила нет — ошибка с подсказкой посмотреть `app_id`/`title` в сообщении и добавить секцию в `~/.config/niri-session/niri-session.conf` или передать `--config /путь`.
