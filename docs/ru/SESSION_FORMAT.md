# Формат файла сессии (JSON)

Файл создаётся командой `niri-session-manage --save` и читается командой `niri-session-manage --load`.

## Версия схемы

Поле **`schema`** (целое число): текущая версия формата — **`1`**. При существенных несовместимых изменениях структуры значение будет увеличено; старые файлы могут перестать читаться без миграции.

## Корневой объект

| Поле | Тип | Описание |
|------|-----|----------|
| `schema` | `number` | Версия формата (см. выше). |
| `niri_version` | `string` | Строка версии niri из IPC (`Request::Version`). |
| `outputs` | `object` | Имена выходов → объекты [`Output`](https://docs.rs/niri-ipc/25.11.0/niri_ipc/struct.Output.html) из `niri-ipc` (как в ответе `Request::Outputs`). |
| `workspaces` | `array` | Снимок рабочих столов на момент сохранения. |
| `windows` | `array` | Окна, которые подлежат восстановлению. |

## Элемент `workspaces[]`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | `number` | Внутренний id рабочего стола в niri (на загрузке **не** используется как ключ). |
| `idx` | `number` | Индекс рабочего стола **на его мониторе** (как в IPC). |
| `name` | `string` \| `null` | Имя рабочего стола, если задано. |
| `output` | `string` \| `null` | Имя выхода (монитора). |

## Элемент `windows[]`

| Поле | Тип | Описание |
|------|-----|----------|
| `command` | `array` of `string` | Аргументы процесса из `/proc/<pid>/cmdline` на момент сохранения (`argv`), с возможными правками (см. ниже). |
| `app_id` | `string` \| `null` | Wayland `app_id`, если есть. |
| `title` | `string` \| `null` | Заголовок окна (для справки). |
| `output` | `string` | Имя выхода, на котором находилось окно. |
| `workspace_idx` | `number` | Индекс рабочего стола на этом выходе (`u8` в IPC). |
| `column` | `number` | Номер колонки в прокручиваемой раскладке, **с 1** (как `pos_in_scrolling_layout.0`). |
| `tile` | `number` | Позиция в стеке колонки, **с 1** (верхняя плитка — 1). |
| `is_floating` | `boolean` | Плавающее окно или тайлинг. |
| `was_focused` | `boolean` | Окно было в фокусе при сохранении; на `--load` используется для возврата фокуса (если поля нет — считается `false`). |
| `column_width` | `number` \| отсутствует | Только тайлы: ширина колонки в **логических пикселях** (ширина из `WindowLayout.tile_size` при сохранении). На `--load` — `SetColumnWidth` после сборки колонки. |
| `window_height` | `number` \| отсутствует | Только тайлы: высота плитки в **логических пикселях** (высота из `tile_size`). На `--load` — `SetWindowHeight` для каждого сопоставленного окна. |

Окна без известного PID или без командной строки в `/proc` при сохранении **пропускаются** (их нельзя восстановить по той же модели).

У **Chrome/Chromium installed PWA** несколько окон могут разделять один PID браузера: в `/proc` в **`--app-id=`** тогда оказывается не тот идентификатор, что у конкретного окна. При **`--save`** и при **`--load`** `niri-session-manage` выравнивает **`--app-id=`** по полю **`app_id`**, если оно вида `chrome-<id>-…` / `chromium-<id>-…` (id — непрозрачная строка до первого `-` после префикса).

Для окон вроде **xwayland-satellite** с **`-listenfd`** в JSON сохраняется фактическая `command` из `/proc` (она непригодна для повторного запуска). При **`--load`** реальный `argv` берётся из TOML [`[[launch]]`](CONFIG.md) по совпадению `app_id` / `title_contains` и поля **`resolve`** (проблемная программа или `-listenfd`). Поля `output`, `workspace_idx`, `column`, `tile`, `is_floating` задают **порядок фокуса** монитора и стола перед каждым запуском. Для **тайлов** необязательные **`column_width`** и **`window_height`** восстанавливают размеры через IPC после расстановки (см. [LOAD_RESTORE.md](LOAD_RESTORE.md)).

## Минимальный пример (фрагмент)

```json
{
  "schema": 1,
  "niri_version": "25.11",
  "outputs": {},
  "workspaces": [],
  "windows": [
    {
      "command": ["foot"],
      "app_id": "foot",
      "title": "foot",
      "output": "HDMI-A-1",
      "workspace_idx": 1,
      "column": 1,
      "tile": 1,
      "is_floating": false
    }
  ]
}
```

Поле `outputs` в реальном файле заполняется полностью; здесь оно сокращено для читаемости.

## Совместимость

- **Версия niri** должна соответствовать зависимости `niri-ipc` в собранном `niri-session-manage`.
- Ручное редактирование `command` допустимо (например, для запуска Flatpak вместо бинаря из `/proc`).
